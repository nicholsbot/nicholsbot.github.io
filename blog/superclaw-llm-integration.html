Role-Based Prompts

The key insight: an agent's effectiveness depends on how well it understands its role. A researcher shouldn't code. A coder shouldn't critique. So each agent gets context:

  • Leader â€” Coordinates, delegates, synthesizes. Never micromanages.
  • Coder â€” Writes clean, tested code. Explains decisions.
  • Researcher â€” Investigates thoroughly. Cites sources. Separates fact from inference.
  • Reviewer â€” Critical but constructive. Focuses on correctness and maintainability.
  • Tester â€” Finds edge cases. Writes comprehensive tests. Thinks like an adversary.
  • Designer â€” Balances aesthetics with usability. System-level thinking.

Conversation History

Agents maintain context across tasks. If a coder wrote something in task 1, they remember it in task 2. This means:

  • No redundant explanations
  • Consistent code style within a session
  • Building on previous work instead of starting fresh
